# 🤝 Contributing to HYPER dOoK Portal

Welcome to the ADHD-first development community! We're excited to have you contribute to this legendary portal.

## 🧠 ADHD-First Development Philosophy

Our development process follows the **BROski LOOK-THEN-BUILD** methodology:

1. **LOOK** 👀: Understand the neurodivergent user needs
2. **BUILD** 🔨: Create with accessibility and celebration in mind
3. **TEST** 🧪: Verify ADHD-friendly functionality
4. **CELEBRATE** 🎊: Every contribution deserves recognition!

## 🚀 Getting Started

### Prerequisites
- Node.js 18.17.0+
- Understanding of ADHD/dyslexia accessibility needs
- Enthusiasm for celebration-driven development!

### Local Setup
\`\`\`bash
git clone https://github.com/welshDog/tHe-HYPER-dOoK-STorY.git
cd tHe-HYPER-dOoK-STorY
npm install
npm run dev
\`\`\`

## 🎯 Contribution Guidelines

### Code Style
- **TypeScript**: Use strict typing
- **Accessibility**: WCAG 2.1 AA compliant
- **Performance**: < 3 second load times for ADHD users
- **Testing**: Include accessibility tests

### Commit Messages
Use semantic commits with celebration:

\`\`\`
✨ feat: add hyperfocus timer with celebration rewards
🐛 fix: improve screen reader navigation in crystal grid  
♿ a11y: add dyslexic font selection option
🎊 celebration: enhance dopamine reward system
📚 docs: add ADHD-friendly setup guide
\`\`\`

### Pull Request Process
1. Create feature branch with descriptive name
2. Include accessibility testing results  
3. Add celebration message for your contribution
4. Request review from ADHD community members

## 🌈 Accessibility Standards

### Required Checks
- [ ] Screen reader compatibility
- [ ] Keyboard navigation support
- [ ] Color contrast 4.5:1 minimum
- [ ] Dyslexic font rendering
- [ ] Reduced motion respect
- [ ] Focus indicators clear and visible

### ADHD-Specific Testing
- [ ] 3-second load time maximum
- [ ] Distraction-free mode functional
- [ ] Celebration system non-overwhelming
- [ ] Clear visual hierarchy
- [ ] Chunked information display
